# handoff checklist

## before handoff
- confirm `status.json` is updated
- confirm `next-step.md` is updated
- confirm key decisions are written into `decision-log.md`
- confirm latest output path is known

## during handoff
- provide `project_id`
- provide current `status`
- provide `current_goal`
- provide `next_action`
- provide blockers
- provide owners / support / assistants

## after handoff
- receiver should read mainline files first
- receiver should reply with structured acceptance, not just "got it"
