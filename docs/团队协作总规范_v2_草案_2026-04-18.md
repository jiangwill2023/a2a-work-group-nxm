## 2. 派单原则

- 调研、资料、分析、会审底稿类任务优先派给 `mr-library`
- 文稿整合、表达优化、对外材料类任务优先派给 `may`
- 网页、小程序、开发、技术校准类任务优先派给 `tracy`
- 涉及预算、盈利、成本、商业判断时，主动拉 `steven` 参与支持
- 如项目需要多人协作，由 `qiang` 负责组建临时项目小组并明确主责和协作关系

## 3. 协作原则

## 5. 共享目录项目结构（标准）

从现在开始，团队默认且唯一的共享项目根目录为：

`/Volumes/Local Drawer/SharedProjects/`

- 所有正式项目必须存放在该目录下

## 11. project_id 与项目文件夹命名规则

### project_id 规则
默认使用：

`p-YYYYMMDD-shortname`

要求：
- 全小写
- 使用英文、数字、连字符 `-`
- `shortname` 应尽量简短、可识别、可搜索
- 不使用空格，不使用中文，不使用随意缩写
- 如原始项目名为中文，落盘前必须先转成英文小写 shortname

示例：
- `p-20260418-a2a-governance`
- `p-20260418-sharedprojects-policy`
- `p-20260418-browser-usage-rule`

### 项目文件夹命名规则
默认要求：
- 项目文件夹名 = `project_id`
- 项目文件夹路径 = `/Volumes/Local Drawer/SharedProjects/<project_id>/`
- 所有落盘目录名、项目根目录名、slug、shortname 一律使用英文小写
- 因共享根路径 `/Volumes/Local Drawer/SharedProjects/` 含空格，所有脚本 / shell 调用必须统一做 quoted path / 安全路径处理

不允许：
- 文件夹名与 `project_id` 不一致
- 同一项目出现多个主线目录
- 使用模糊名称如 `test`、`new-project`
- 直接使用中文目录名作为正式项目主线目录

## 12. status / decision 更新规则

### status.json 维护责任
- 默认由 `pm_owner` 负责维护

- deadline 变化
- 主责角色变化

- 用户提出新的关键要求
- blocked 后改走新路径

## 13. A2A 与项目文件结合规则

- A2A 消息负责发起、通知、ACK、BLOCKED
- 项目文件负责承接任务、记录决策、沉淀进度、保存证据
- 不允许只有聊天没有文件
- 不允许只有文件没有当前状态回执
- 每次分派后，后续推进应尽快落到对应项目文件夹

### 正式派单建议字段
每条正式派单建议至少带：
- `project_id`
- `task_id`
- `pm_owner`
- `file_path`
- `expected_reply`

推荐格式：

```text
project_id=<project_id>
task_id=<task_id>
file_path=/Volumes/Local Drawer/SharedProjects/<project_id>/
请先读取项目文件，再执行。
请仅回复 ACK 或 BLOCKED。
```

### 回执状态解释
- `started` = 已发起，不等于闭环
- `timeout` = 当前等待窗口内未拿到即时回执，不等于失败
- `received` = 后续证据已确认对方收到了，但尚未证明 `ACK / BLOCKED / 已执行`
- `ACK` = 已收到、已确认
- `BLOCKED` = 已收到但无法执行，需要显式说明阻塞
- 不允许把 `timeout` 直接误报为失败；若后续确认已收到，应将状态修正为 `received`

## 14. 项目创建 SOP（标准流程）

从下一项目开始，按以下顺序执行：
1. `qiang` 判断是否立项
2. 判断任务类型并决定 `pm_owner`
3. 生成 `project_id`
4. 在 `/Volumes/Local Drawer/SharedProjects/` 下创建项目文件夹
5. 初始化：
   - `00-brief/brief.md`
   - `01-status/status.json`
   - `01-status/next-step.md`
   - `02-decisions/decision-log.md`
6. 在 `brief.md` 和 `status.json` 中写入初始信息
7. `status.json` 初始状态设为 `planning` 或 `ready`
8. 再发起 A2A 协作与派单
9. 回执时以项目文件中的最新内容为准
10. 如发现文件散落在非共享根目录，应明确标记为未挂主线，并尽快迁回主线目录

## 16. 异常升级机制

- `pm_owner` 卡住或长期无推进 → 升级到 `qiang`
- 技术判断不确定 / 技术质量风险 → 升级到 `tracy`
- 资料不足 / 分析不稳 / 会审底稿不扎实 → 升级到 `mr-library`
- 文稿表达不成型 / 对外材料不够顺 → 升级到 `may`
- 预算 / 成本 / 盈利 / 商业判断不清 → 升级到 `steven`
- `coder` 执行失速或质量不稳 → `mr-library` 先管，`tracy` 做技术把关
- `leon` 执行失速或质量不稳 → `may` 先管，`tracy` 做技术把关
